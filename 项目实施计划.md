# 家政服务系统 — 项目实施计划

## 一、项目概述

### 1.1 项目基本信息

| 项目名称 | 基于Spring Boot+Vue的家政服务系统的设计与实现 |
|----------|---------------------------------------------|
| 架构模式 | B/S架构，前后端分离 |
| 开发模式 | 敏捷开发，迭代式交付 |
| 预计周期 | 4-6周 |

### 1.2 项目目标

构建一个完整的家政服务管理平台，实现：
- 用户在线预约家政服务
- 服务人员接单管理
- 管理员后台运营管理
- 完整的订单生命周期管理
- 评价与信誉体系

## 二、需求分析总结

### 2.1 用户角色

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| 普通用户 | 需要家政服务的客户 | 浏览服务、预约下单、订单管理、评价服务 |

| 模块 | 功能点 | 优先级 |

| 服务预约 | 服务详情、选择时间、填写地址、提交订单 | P0 |
| 订单管理 | 订单列表、订单详情、取消订单、确认完成 | P0 |
| 评价管理 | 星级评分、文字评价、历史评价 | P1 |

#### 服务人员端（优先级：中）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|

| 接单管理 | 新订单通知、接单/拒单 | P0 |
| 工作台 | 进行中订单、历史订单、收入统计 | P1 |

#### 管理后台（优先级：高）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 用户管理 | 用户列表、搜索、启用/禁用 | P0 |
| 服务人员管理 | 列表、审核、信息管理 | P0 |
| 服务类目管理 | 增删改查、上下架 | P0 |
| 订单管理 | 列表、筛选、派单、异常处理 | P0 |
| 评价管理 | 列表、删除违规评价 | P1 |
| 数据统计 | 用户量、订单量、营收统计 | P1 |

## 四、数据库设计概要

### 4.1 核心数据表

| 表名 | 说明 | 记录数预估 |
|------|------|-----------|
| sys_user | 用户表（含管理员） | 10000+ |
| service_category | 服务类目表 | 20+ |
| worker | 服务人员表 | 500+ |
| orders | 订单表 | 50000+ |
| review | 评价表 | 30000+ |
| address | 地址表 | 15000+ |

### 4.2 关键索引

```sql
-- 用户表索引
CREATE INDEX idx_user_phone ON sys_user(phone);
CREATE INDEX idx_user_role ON sys_user(role);

-- 订单表索引
CREATE INDEX idx_order_user ON orders(user_id);
CREATE INDEX idx_order_worker ON orders(worker_id);
CREATE INDEX idx_order_status ON orders(status);
CREATE INDEX idx_order_time ON orders(create_time);

-- 评价表索引
CREATE INDEX idx_review_worker ON review(worker_id);
```

## 五、项目结构

## 七、质量标准

### 7.1 代码质量

- 遵循Java和Vue编码规范
- 关键代码添加注释
- 代码复用率 > 60%
- 无严重代码警告

### 7.2 性能指标

| 指标 | 标准 |
|------|------|
| 页面加载时间 | ≤ 3秒 |
| API响应时间 | ≤ 500ms |
| 数据库查询 | ≤ 200ms |
| 并发用户数 | ≥ 100 |

### 7.3 安全要求

- JWT Token认证
- 密码BCrypt加密
- SQL注入防护
- XSS攻击防护
- RBAC权限控制

### 7.4 用户体验

- 界面简洁美观
- 操作流程直观
- 错误提示明确
- 响应式设计

## 十、开发规范

### 10.1 命名规范

**后端：**
- 类名：大驼峰（UserService）
- 方法名：小驼峰（getUserById）
- 常量：全大写下划线（MAX_SIZE）
- 包名：全小写（com.example.homeservice）

**前端：**
- 组件名：大驼峰（UserList.vue）
- 变量名：小驼峰（userList）
- 常量：全大写下划线（API_BASE_URL）

### 10.2 Git提交规范

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具相关
```

### 10.3 接口规范

- RESTful风格
- 统一响应格式
- 统一错误码
- 版本控制（/api/v1/）
