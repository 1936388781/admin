# 家政服务系统 — 系统设计文档

## 课题名称：基于Spring Boot+Vue的家政服务系统的设计与实现

---

## 一、引言

### 1.1 编写目的

本文档对家政服务系统进行详细的系统设计，包括系统架构设计、功能模块设计、数据库设计、接口设计等内容，为后续的编码实现提供技术指导。

### 1.2 设计原则

- **分层架构**：采用Controller-Service-Mapper三层架构，职责清晰
- **前后端分离**：前端Vue + 后端Spring Boot，通过RESTful API通信
- **高内聚低耦合**：各模块功能独立，减少模块间的依赖
- **可扩展性**：预留接口和扩展点，便于后续功能迭代
- **安全性**：JWT认证 + RBAC权限控制 + 数据加密

## 四、数据库设计

### 4.1 E-R图（文字描述）

**核心实体及关系：**

```
用户(1) ──────< 订单(1) ──────> 服务类目(1)
  │               │
  │               │
  └──────< 评价(1)
                  │
服务人员(1) ──────┘
  │
  └──────< 地址(1)

管理员(1) ──────> 管理所有实体
```

**关系说明：**
- 一个用户可以创建多个订单，一个订单属于一个用户（1:N）
- 一个订单关联一个服务类目，一个服务类目可被多个订单引用（1:N）
- 一个用户可以对多个订单进行评价（1:N）
- 一个服务人员可以接收多个订单，一个订单分配给一个服务人员（1:N）
- 一个服务人员可以收到多个评价（1:N）
- 一个用户可以保存多个地址（1:N）

### 4.3 建表SQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS homeservice DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE homeservice;

-- 用户表
CREATE TABLE sys_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码',
    phone VARCHAR(20) NOT NULL UNIQUE COMMENT '手机号',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar VARCHAR(255) COMMENT '头像URL',
    role TINYINT NOT NULL DEFAULT 0 COMMENT '角色：0-普通用户 1-服务人员 2-管理员',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-禁用 1-启用',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB COMMENT='用户表';

-- 服务类目表
CREATE TABLE service_category (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    name VARCHAR(50) NOT NULL COMMENT '类目名称',
    description VARCHAR(500) COMMENT '类目描述',
    price DECIMAL(10,2) NOT NULL COMMENT '参考价格（元/次）',
    icon VARCHAR(255) COMMENT '类目图标URL',
    sort_order INT DEFAULT 0 COMMENT '排序序号',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-下架 1-上架',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB COMMENT='服务类目表';

-- 服务人员表
CREATE TABLE worker (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    user_id BIGINT NOT NULL UNIQUE COMMENT '关联用户ID',
    real_name VARCHAR(50) NOT NULL COMMENT '真实姓名',
    gender TINYINT COMMENT '性别：0-女 1-男',
    age INT COMMENT '年龄',
    phone VARCHAR(20) COMMENT '联系电话',
    id_card VARCHAR(20) COMMENT '身份证号',
    skills VARCHAR(500) COMMENT '技能标签（JSON数组）',
    experience INT DEFAULT 0 COMMENT '从业年限',
    level TINYINT DEFAULT 1 COMMENT '等级：1-初级 2-中级 3-高级 4-金牌',
    rating DECIMAL(2,1) DEFAULT 5.0 COMMENT '综合评分',
```

## 五、接口设计

### 5.1 接口规范

**统一响应格式：**

```json
{
    "code": 200,
    "message": "操作成功",
    "data": { }
}
```

**状态码定义：**

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/Token过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

```json
{
    "code": 200,
    "message": "操作成功",
    "data": {
        "records": [],
        "total": 100,
        "size": 10,
        "current": 1,
        "pages": 10
    }
}
```

## 六、安全设计

### 6.1 认证流程

```
用户登录请求
    │
    ▼
AuthController.login()
    │
    ▼
验证用户名密码
    │
    ├── 失败 → 返回错误信息
    │
    └── 成功 → 生成JWT Token
                │
                ▼
          返回Token给前端
                │
                ▼
        前端存储Token（localStorage）
                │
                ▼
        后续请求携带Token（Header: Authorization: Bearer <token>）
                │
                ▼
        JwtAuthenticationFilter 验证Token
                │
                ├── 无效/过期 → 返回401
                │
                └── 有效 → 放行请求，设置用户上下文
```

### 6.2 权限控制

| 角色 | 可访问模块 |
|------|-----------|
| 普通用户（role=0） | 首页、服务浏览、预约下单、订单管理、评价、个人中心 |
| 服务人员（role=1） | 个人信息、接单管理、工作台 |
| 管理员（role=2） | 管理后台所有功能 |
